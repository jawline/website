Scribble - 2013 / 2015

For my university dissertation I designed and implemented a programming language and virtual machine.

The language was designed to make extending existing C++ applications easier and to enable users to extend a programs functionality, increasing the usefulness of a piece of software. The implementation also attempts to reduce the likelihood of security issues by allowing developers to strictly control what functionality the virtual machine exposes to the end user through a simple interface, allowing them to control what access the scripts will have to the host platform.

Scribble is strictly typed but supports a simple type inference system which allows for variable types to be inferred from their initial declaration, allowing for the benefits strictly typed languages provide to large projects without too√ü much additional typed overhead. It was originally designed as a procedural language with a simple C like syntax. However as it has grown I have begun to extend it with a simple, experimental, functional style syntax which I hope to extend in the future (this is available in the feature/language_experiments branch on Github).

The language and accompanying tools are primarily written in C++, the repository also contains a set of examples written in Scribble (Note: Several of these examples are not compatible with the version of the code in the feature/language_experiments branch).

Since leaving university I have continued to update the project.

More information and the full source code for the project is available here

Kiniro - 2013 / 2015

For the last couple of years I have been developing a simple multi platform game engine and a game to go along with it.

The project does not use any pre-built engine and handles the entire game lifecycle including asset management, rendering, audio, physics, actor logic, game world logic, and all of the other features which are commonly handled by the engine.

The aim of the project was not to create the best game, but to get a firm understanding of the entire logic and lifecycle of a game engine and to try and build a solid foundation from which developing a new game would be made easy.

The core engine is written in C++, using libRocket for UI, OpenGL for rendering, OpenAL for audio and Freetype for text. Several other tools have been written to assist with developing on the engine, such as a file format exporter, the majority of these tools are written in python.

The game engine currently supports Linux and Android.

DawnOS - 2010 / 2015

Between my school years of twelve and thirteen I developed a x86 based operating system (OS) with a process scheduler, virtual memory manager, application loader, a defined set of system calls and a host of other features. This project was for the OCR extend project A level qualification which I completed in 2011.

The project involved development of a operating system kernel and a set of utility programs, as well as the design of a set of tools designed for the OS which allow the user to interact and perform tasks. The code was primarily written in C (C99), with small amounts of assembly used in the Kernel, as well as some bash or python scripts to automate compilation and deployment of the OS.

In 2014 I began working on the project again, refactoring the older code so that it complies with more modern programming standards and adding new features. I am continuing to work on the project, along with one other developer, to improve the existing code and experiment with potential kernel and OS designs.

The project is now available on github here

SmoothHome - 2013

SmoothHome is a minimal Android home screen replacement meant to hide the complexity of a smart phone and give you easy access to the things you use most.

SmoothHome was developed in Java for the Android platform. The project involved learning how to handle the life cycle of a application which is always running. It also required me to develop my knowledge of the Android SDK in order to create manifests which appropriately declared Activities in such a way that the Android operating system would see them as a home screen replacement.

Cool Computing Competition - 2011

I received a prize for the development of two applications which I entered into Royal Holloway's cool computing competition.

The first was a application that could detect and store any faces that were placed in front of the camera, the idea being that it would allow an automated way to store records of people in a room, and perhaps extensible to the degree that it could be used to identify and record objects within a room (I.e. take a picture of every bottle in a room).

The second was a 3D engine targeted at the Android operating system. The idea was to create an easy way to export and display models created in the Blender modelling program on an Android phone and use them in 3D applications such as games. The project involved a toolchain that could convert blender models to a custom internal format capable of being loaded on Android. It could also compress and modify image files using the Zip file format which would reduce the file size of the application. The source code of the application was written in Java whilst the toolchain was written in Python.

Both of these applications ran on Android 2.1 and up - the latter used OpenGL ES 2.0
